Troubleshooting ComiCat

I am getting sync failures

There are a few possible reasons why a file may fail to sync

  • The file is password or DRM protected.
  • The file is incomplete, it happens when resync is done while the file is still being copied/downloaded.
  • The file is on external sdcard and the sdcard is encrypted.
  • The file has been saved offline from Google drive. Google drive keeps offline files encrypted which makes them unreadable for other apps. Turn off encryption from Google drive's settings and then do a re-sync

A lot of non comic files appear on my shelf

This can happen when ComiCat is set to scan for secondary comic file extensions like zip and pdf. Since these are general purpose formats, found files can be non comic files. You can try following

  • If the number of files is limited you can 'Exclude' them from catalog. This can be done using the context menu in catalog view.
  • You can set "Include Zip, RAR, PDF format files" preference in "Catalog Settings" to "Don't Include" or "Include only if inside comic folder".

Some of my comics are not scanned by ComiCat and I don't get any sync failure

It means ComiCat ignored those comic files. Besides the primary comic format extensions ComiCat can look for .zip, .rar, .pdf extensions and import them to your catalog. But which files it will pick will depend on "Include Zip, RAR, PDF format files" settings. You can read about this setting in the "Catalog Settings" section of "Settings Guide". You can either change this setting to the desired value or move the comics to an appropriate folder to have them imported.

My comics look blurry

Try increasing the "Max Image Size" setting in "Performance Settings". We recommend that you increase the value in 1MB steps until you get crisp images.

I get force close while reading comics

This may either be because of a corrupt Comic file or memory issues. If it happens only with one file then most probably it is the former else you can try the following

  1. In performance settings reduce the value of "Max Image Size" setting.
  2. Turn off "Aggressive Caching".
  3. If the above two don't work then turn off "Show Transitions"

I deleted and now want to re-download a comic from dropbox but ComiCat won't do it.

By default ComiCat maintains and uses download history to avoid re-downloading deleted files. This is so that the users can freely delete Comic files from their device once they are done reading it. You can turn this off from "Cloud Sync Settings". Please note, turning off this setting does not clear the download history, it only tells ComiCat to ignore it. You can re-enable the setting any time without losing the download history.

ComiCat insists I see double page spreads as single page

By default ComiCat is set to split Double Page Scans and show them as single pages. If this is not desired you can turn it off from "Double Page Scans" setting in viewer settings.
